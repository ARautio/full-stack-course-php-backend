Databases

Introduction Databases

Databases means a set of data which is organised certain way so everything from excel sheets to complex set of data organisation can be called as databases. In this course and mostly in web development, a relational database is used.

If to simplify relational databases are a bit like an excel file. They can contain multiple tables and each table can consist from rows and columns.

Column types in tables

  • Text: varchar
  • Numeric: integer, float, decimal
  • Date and time: date, datetime, time, timestamp
  • Options: auto increment, unique, primary key

Text: Varchar

Varchar is like string PHP but since we are working with structured data, we also need to know the maximum size of the string. That's why varchar needs a max size.

Numeric: integer, float, decimal

For numerical presentation we have two same datatypes as with PHP (integer and float). The only difference with PHP is that integer can be defined to have only positive number by giving it UNSIGNED attribute.

Besides already familiar datatypes, we have Decimal which is more simply saying more defined float. Decimal will always have certain amount of numbers after dot. This is very handly for monetary values or other clearly defined numeric values.

Date and time: date, datetime, time, timestamp

Date and time datatypes are to store specific dates and times. They can be used to save date of events like when the feedback has been created. Or they can be used to save a start and end date for example for reservations.

  • Date datatype defines only the date like 15.10.2016.
  • Datetime defines both date and time like 15.10.2015 22:30
  • Timestamp is a numeric value which counts seconds from date XXXX.

We go through date functions more in couple of weeks.

auto increment, unique, primary key

Besides the datatypes there are attributes for each column which can be used and they come handy in certain situations.

  • Auto increment is for columns with datatype integer. For every row it adds a bigger number. Like first row will get 1, second 2, third 3 and so on. The start number if auto increment can be also defined. This is very good for example order number.
  • Unique is for all columns in general. You can define that certain column or columns needs to be unique when inserting a row and if they are not the database will give an error. This is good for example username.
  • Primary key is column which is set to be unique key for the rows. It makes searches faster when there are huge amoun to data.
